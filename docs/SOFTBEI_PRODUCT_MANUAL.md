# 软件杯产品说明书

## 1. 产品简介

`设备检修知识与作业助手` 是一套面向设备检修场景的知识检索与标准化作业辅助系统。系统以摩托车发动机检修为当前演示对象，支持通过文本、设备型号和故障图片检索知识，并将知识结果转化为可执行的检修任务，同时支持案例沉淀、审核与知识回流。

## 2. 使用对象

- 检修技师
- 审核人员
- 比赛评委和答辩老师

## 3. 系统入口

- 正式入口：`softbei_workbench.html`
- 子模块：
  - `knowledge_search.html`
  - `maintenance_tasks.html`
  - `case_reviews.html`
  - `diagnosis_console.html`

## 4. 功能说明

### 4.1 知识检索

- 输入检修问题、设备型号，必要时上传一张故障图片
- 查看命中的知识条目、来源文件、页码和推荐理由

### 4.2 检修任务

- 从检索结果生成标准化检修任务
- 按步骤执行、补充备注并完成确认
- 导出任务摘要和引用来源

### 4.3 案例沉淀

- 将检修过程和结果提交为案例
- 对检索结果和总结进行人工修正
- 审核通过后回流知识库

### 4.4 智能分析辅助

- 查看大模型给出的辅助分析与解释
- 该模块为辅助展示，不替代知识检索与作业主链路

## 5. 推荐操作流程

1. 打开正式工作台
2. 在知识检索区输入固定检索词
3. 选择命中知识并创建检修任务
4. 执行步骤并导出摘要
5. 提交案例并演示审核回流

## 6. 固定演示建议

- 设备类型：`摩托车发动机`
- 设备型号：`LX200`
- 固定检索词：
  - `火花塞`
  - `冷启动困难`
  - `正时链条`
  - `机油渗漏`
  - `温度偏高`

## 7. 常见问题

### 7.1 搜不到知识条目

- 先确认设备型号是否填写正确
- 优先使用固定检索词
- 先验证知识库是否已导入对应手册或案例

### 7.2 图片识别不稳定

- 当前多模态为增强入口
- 正式演示优先使用“文本 + 型号”链路

### 7.3 智能分析响应慢

- 可先完成知识检索和作业闭环主流程
- 智能分析只作为补充说明

## 8. 当前版本说明

当前版本已满足软件杯主链路演示需求，但仍需配套 PPT、视频、正式提交包和最终答辩资料。
