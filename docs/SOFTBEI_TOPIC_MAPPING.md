# 软件杯赛题映射矩阵

## 选题信息

- 赛题：`A基于多模态大模型技术的设备检修知识检索与作业系统`
- 赛题页：<https://www.cnsoftbei.com/content-15-1246-1.html>
- 当前作品定义（冻结版）：`设备检修知识与作业助手`

## 作品重定义

当前仓库不再以“工业故障诊断系统”作为软件杯主产品定义，而改为：

> 一套面向检修技师、班组长和知识管理员的设备检修知识与作业助手，支持多模态知识检索、标准化检修指引、案例沉淀与审核回流，并保留现有诊断链路作为智能分析子模块。

## 冻结的演示对象与用户角色

- 演示对象：`摩托车发动机检修`
- 核心设备：`发动机总成`
- 典型故障：`启动困难`、`异常异响`、`温度异常`
- 用户角色：
  - 检修技师：查询知识、执行作业、上传案例
  - 班组长：复核步骤、查看历史任务
  - 知识管理员：审核案例、修正结果、更新知识

## 赛题要求与当前实现差距

| 赛题要求 | 当前项目已有 | 主要缺口 | 目标落地形态 |
|----------|--------------|----------|--------------|
| B/S 架构 + Web/App 界面 | FastAPI + 静态 Web 页 | 当前界面是调试台，不是正式系统 | 正式 Web 界面，固定演示入口 |
| 支持本地或云端大模型服务 | 已支持 DeepSeek / OpenAI 兼容接口 / Anthropic | 缺少与检修知识系统绑定的主业务流 | 将 LLM 嵌入检索总结、图片识别、作业建议 |
| 多模态知识检索 | 仅有文本症状和时间窗输入 | 没有知识库、型号检索、图片输入 | 文本/型号/图片联合检索 |
| 快速调取检修手册等资源 | 无知识文档入库 | 没有手册切片、索引和出处引用 | 文档导入、全文检索、来源页码返回 |
| 标准化作业指引 | 仅有诊断报告文本 | 没有步骤卡、风险提示、任务状态 | 检修任务 + 步骤模板 + 合规提醒 |
| 个性化推送流程 | 无设备类型/检修等级模板 | 任务模型缺失 | 按设备类型和检修等级生成步骤 |
| 知识沉淀与更新 | 无案例上传与审核 | 无案例回流、人工纠错、知识关系 | 案例上传、审核、入库、后续命中 |
| 手动标注与修正模型结果 | 无 | 无人工修正入口 | 检索/建议结果勘误与采纳记录 |
| 软件界面美观、易用、稳定 | 当前后端稳定、SSE 可演示 | 前端完成度不足 | 正式前端 + 固定演示主线 |
| 完整文档与材料 | README、部署文档、demo checklist | 缺需求分析、设计文档、说明书、测试报告、PPT、视频 | 竞赛提交包 |

## 当前可复用能力

- FastAPI 后端与 PostgreSQL 基础能力
- 现有日志、部署、CI、测试骨架
- 当前多智能体诊断链路
- SSE 过程展示
- 工业设备场景叙事能力

## 当前必须降级的能力定位

以下能力保留，但不能继续作为作品主体：

- `POST /api/v1/diagnose`
- `GET /api/v1/diagnose/stream`
- 基于时间窗的传感器故障诊断

这些能力后续定位为：

- “知识检索后的智能分析建议”
- “检修任务中的辅助诊断面板”
- “扩展演示能力，而不是主产品定义”

## 当前排除项

- `LoongArch + 银河麒麟高级服务器版` 部署适配

该项是赛题硬性要求，但当前阶段不纳入主线开发，后续若进入正式提交阶段必须单独补齐。
