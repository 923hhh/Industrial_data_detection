# 部署与运维文档

## 1. 文档目的

记录本仓库 **工业故障检测后端** 与 **检修域 MVP** 的部署方式、依赖、环境变量与健康检查；并给出 **龙芯 LoongArch + 银河麒麟** 目标环境的验证清单（与《业务场景定义文档》对齐）。

## 2. 常规部署（x86 / Windows 或 Linux）

1. Python 3.10+，创建 venv，`pip install -r requirements.txt`。
2. 配置 `.env`：`DATABASE_URL`（开发可用 `sqlite+aiosqlite:///./sensor_data.db`，生产建议 PostgreSQL + `asyncpg`）。
3. `alembic upgrade head`。
4. （可选）检修域演示账号：`PYTHONPATH=. python scripts/seed_mvp_domain_users.py`。
5. 启动：`uvicorn app.main:app --host 0.0.0.0 --port 8000`。
6. 健康检查：`GET /health`；检修子域：`GET /api/v1/maintenance/health`。

### 2.1 检修域环境变量（补充）

| 变量 | 说明 |
| ---- | ---- |
| `JWT_SECRET_KEY` | JWT 签名密钥，**生产须为强随机串（建议 ≥32 字节）** |
| `JWT_ALGORITHM` | 默认 `HS256` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | 访问令牌有效期（分钟） |
| `MAINTENANCE_UPLOAD_DIR` | 检修附件本地存储目录 |
| `ATTACHMENT_SIGN_SECRET` | 附件短时下载链接 HMAC 密钥 |

## 4. 运维要点

- **日志**：HTTP 中间件已打 request_id；检修域关键操作写入 `audit_logs` 表。
- **备份**：生产应对 PostgreSQL 做定期快照；附件目录 `MAINTENANCE_UPLOAD_DIR` 需纳入备份或对象存储迁移计划。
- **密钥**：API Key 仅走环境变量，勿写入 `system_configs` 明文（见《数据字典》`system_configs` 策略）。
