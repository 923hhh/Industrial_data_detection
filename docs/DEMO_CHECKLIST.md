# 真实浏览器联调清单

该清单用于比赛演示前的最终核对，重点是验证“前端页面 + 后端接口 + 真实模型调用”是否已经跑通。

## 演示前准备

- `.env` 中已填入真实可用的 API Key
- 数据库已初始化
- 后端服务已启动
- 浏览器能直接打开 `index.html`

## 页面联调检查

### 基础可用性

- 页面能正常打开
- “后端地址”会自动读取浏览器本地保存值
- 留空后端地址时，会自动回退到默认逻辑
- 点击“清空”后界面状态恢复

### 输入校验

- 起始时间为空时会阻止请求
- 结束时间为空时会阻止请求
- 结束时间早于起始时间时会阻止请求
- 非法后端地址会在前端直接报错

### 后端不可用场景

- 后端未启动时，页面能给出明确错误提示
- 后端地址错误时，页面能提示检查地址和服务状态
- 连接超时时，页面会给出可理解的错误说明

### 真实链路场景

使用真实时间窗口和真实模型调用，确认顺序为：

1. `connected`
2. `node_start`
3. `node_finish`
4. `report`
5. `done`

同时确认：

- 报告内容完整
- 中文显示正常
- 连接结束后按钮重新可点击

## 后端联调检查

- `/health` 返回正常
- `/docs` 可访问
- `/api/v1/diagnose` 可返回完整报告
- `/api/v1/diagnose/stream` 可持续输出事件流
