## 2. 固定演示对象

- 设备类型：`摩托车发动机`
- 设备型号：`LX200`
- 主线案例：`SB-01 启动困难积碳复盘案例`

## 3. 推荐演示顺序

1. 打开正式工作台，展示四个业务区：
   - 知识检索
   - 检修任务
   - 案例沉淀
   - 历史记录
2. 使用 `SB-01` 做检索演示：
   - 查询词：`发动机启动困难，火花塞积碳明显`
   - 设备类型：`摩托车发动机`
   - 设备型号：`LX200`
3. 展示引用来源：
   - `engine_ignition_manual.pdf`
   - `第2章 / P12`
4. 从检索结果进入标准化检修任务
5. 展示步骤完成和导出摘要
6. 展示案例沉淀与审核回流
7. 需要补充“智能分析”时，再打开子模块页，不把它作为主页面

## 4. 模型与配置口径

- 自动评测固定口径：`文本 + 设备型号`
- 现场主演示推荐：
  - `model_provider=deepseek`
  - `model_name=deepseek-chat`
- 若现场网络不稳定：
  - 知识检索、任务闭环、案例回流优先展示
  - 图片识别和 SSE 智能分析改为可选补充环节

## 5. 失败兜底话术

### 图片识别不稳定

使用：

- 文本描述
- 设备型号

继续演示主链路，并说明：

`多模态入口已经实现，但现场优先展示稳定的文本+型号主路径。`

### 智能分析子模块响应慢

说明：

`当前主产品是知识检索与作业系统，SSE 智能分析模块是辅助能力，不影响主流程展示。`

### 检索结果不符合预期

切回固定案例：

- `SB-01`
- `SB-04`

并避免现场自由输入随机故障。

## 6. 演示前检查

- 后端已启动
- `GET /health` 正常
- 正式工作台可打开
- 当前数据库已执行迁移
- 当前知识库已有固定演示数据
- 当前 API Key 可用
- 当前公网地址或本地后端地址已写入页面

## 7. 演示禁忌

- 不现场临时更换设备对象
- 不把旧版诊断控制台当主页面
- 不现场尝试知识域外问题做主要展示
- 不把长段大模型报告当唯一输出
