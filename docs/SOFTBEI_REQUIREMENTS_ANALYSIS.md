# 软件杯需求分析文档

## 1. 项目概述

### 1.1 项目名称

`设备检修知识与作业助手`

### 1.2 赛题对应

- 赛题名称：`A基于多模态大模型技术的设备检修知识检索与作业系统`
- 赛题来源：第十五届中国软件杯
- 当前聚焦对象：`摩托车发动机检修`

### 1.3 项目定位

本项目不是通用问答系统，也不是单一故障诊断 demo，而是一套面向设备检修场景的知识检索、标准化作业指引与案例沉淀系统。现有智能分析链路作为辅助模块保留，用于增强检索后的判断与解释。

## 2. 需求背景

### 2.1 行业痛点

- 检修知识分散在手册、经验记录和口头传授中，检索成本高。
- 一线检修人员遇到故障时，往往只能凭经验判断，标准化不足。
- 大模型直接回答虽快，但缺少知识出处和作业闭环，难以直接用于实际检修。
- 一次检修的经验难以结构化沉淀，导致相似问题反复排查。

### 2.2 建设目标

- 让检修人员能通过文本、设备型号、故障图片快速检索到相关知识。
- 将知识检索结果转化为标准化作业步骤，而不仅是文本建议。
- 将每次检修案例沉淀为可审核、可复用的新知识。
- 形成“检索 -> 作业 -> 沉淀 -> 回流”的闭环。

## 3. 用户与场景

### 3.1 用户角色

- 检修技师：发起知识检索、执行检修任务、提交案例。
- 审核人员：复核案例内容、修正总结、审核入库。
- 演示评委：快速理解系统价值，关注功能完整性、稳定性和实用性。

### 3.2 核心场景

- 场景 A：设备出现启动困难、异响、渗漏等故障时，检修技师快速检索知识。
- 场景 B：根据检索结果自动生成标准化作业指引并执行。
- 场景 C：检修完成后录入实际处理过程和结果，进入审核与回流。

## 4. 功能需求

### 4.1 知识检索

- 支持文本、设备型号、单张故障图片作为输入。
- 检索结果需返回命中知识条目、来源文件、章节或页码。
- 支持通用手册与具体型号知识共同参与检索。

### 4.2 标准化作业指引

- 根据检索结果生成检修任务。
- 任务包含步骤编号、操作说明、风险提示、注意事项和完成确认。
- 每一步都能回溯到知识来源。

### 4.3 案例沉淀与审核

- 支持录入故障描述、处理步骤、处理结果、附件信息。
- 支持人工修正检索结果和总结。
- 审核通过后自动回流入知识库。

### 4.4 智能分析辅助

- 在知识检索和作业流程之外，提供大模型辅助分析。
- 该模块不是主产品入口，只作为辅助说明与排查建议来源。

## 5. 非功能需求

- 演示稳定：固定案例和固定检索词下需稳定命中。
- 界面清晰：评委需在 3 到 5 分钟内理解系统流程。
- 可追溯：知识检索、作业步骤和案例回流都要有明确来源。
- 可部署：支持 PostgreSQL + FastAPI 的独立部署。

## 6. 边界与排除项

- 当前版本不追求完整工业协议接入。
- 当前版本不做复杂角色权限体系。
- 当前版本暂不纳入 `LoongArch + 银河麒麟` 适配。
- 当前版本不做视频、语音等更重的多模态扩展。

## 7. 需求覆盖矩阵

| 赛题核心方向 | 当前系统实现 |
| --- | --- |
| 多模态知识检索 | 已支持文本、设备型号、单图输入 |
| 标准化作业辅助 | 已支持任务生成、步骤执行、导出摘要 |
| 知识沉淀更新 | 已支持案例上传、修正、审核回流 |
| 大模型辅助能力 | 已保留智能分析子模块 |

## 8. 当前阶段结论

当前项目已完成软件杯核心主链路，但正式提交前仍需收口材料、PPT、视频、截图与部署证明。该文档作为需求层源文件，用于后续功能设计、产品说明书、PPT 和演示视频保持一致。
