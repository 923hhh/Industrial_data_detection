# 📄 软件功能设计与工程优化方案

**项目名称：** 基于多模态大模型技术的设备检修知识检索与作业系统 **应用场景：** 工业制造领域（如：钢铁、汽车制造、发动机维修）设备检修 **运行环境要求：** LoongArch 架构、银河麒麟 V11/V10、4核 CPU、8GB 内存

## 1. 核心架构设计：垂直领域增强型组合

针对工业设备检修场景中“容错率低、实体关联复杂、跨模态查询多”的痛点，本系统采用 **“知识图谱 + 混合检索 + 反馈进化”** 的重型 RAG（检索增强生成）底座设计，分为以下四个关键模块：

### 1.1 知识预处理：告别硬切分，建立层级与图谱

- **语义切分与标题增强：** 摒弃固定字数切分。通过解析《摩托车发动机维修手册》等技术文档的目录树，按“章-节-故障现象”的自然逻辑边界进行语义切块（Semantic Chunking），并为每个数据块拼接层级上下文（Chunk Headers）。
- **知识图谱 RAG (Graph RAG)：** 针对复杂的设备联动关系（如“传感器-控制单元-执行器”），抽取实体与关联，构建设备故障树知识图谱，解决跨组件的复杂故障溯源。

### 1.2 检索阶段：双轨并行应对复杂输入

- **混合检索引擎 (Hybrid Search)：**
	- **BM25 关键词检索：** 负责精准匹配设备型号、故障代码（如“E204”）、特定零件号。
	- **向量检索：** 负责理解一线工人模糊的、口语化的症状描述（如“发动机哒哒响”）。
	- 二者结合，大幅消除检索盲区。

### 1.3 结果组装与输出：严守合规性

- **小块查，大块答 (Small to Big Retriever)：** 检索时利用细粒度的症状描述命中目标，但在组装提示词（Prompt）交给大模型时，自动加载该节点对应的完整操作规程（SOP）或整页手册，确保输出的操作步骤连贯且合规，严防断章取义。

### 1.4 系统进化：业务闭环机制

- **用户反馈循环 (Feedback Loop)：** 系统预留纠错与标注接口。一线人员修正后的正确诊断方案，将作为新节点实时补充进本地图谱，并可作为高质量的 SFT（指令微调）数据沉淀，实现系统越用越聪明。

## 2. 落地风险与难点评估 (风险预警)

在赛题指定的严苛硬件限制（**8GB 内存、无独立 GPU**）下，上述架构面临以下严峻挑战：

1. **极端硬件资源挤压 (OOM 风险)：** 同时在 8GB 内存中运行大语言模型、图数据库、向量数据库和后端服务，极易导致内存溢出或系统崩溃。
2. **知识图谱工程量大：** 从非结构化 PDF（包含表格、流程图）中准确抽取实体和关系，难度极高。大模型若抽取错误（Garbage In），将直接导致图谱检索失效（Garbage Out）。
3. **响应延迟极高：** 链路过长（意图识别 -> 查图谱 -> 查向量 -> 排序融合 -> 生成），在普通 4 核 CPU 上可能导致单次问答耗时过长，严重影响用户体验。
4. **混合检索调参玄学：** 向量得分与 BM25 词频得分标准不一，需要大量实验设定权重与融合算法（如 RRF），调试成本高。

## 3. “减重”与工程优化方案 (比赛突围策略)

为确保系统在国产化软硬件平台上稳定演示并获得高分，特制定以下“减重”与架构优化策略：

### 3.1 算力分离：全面拥抱云端 API

- **策略：** 严格遵循“本地轻量化，重算力上云”的原则。
- **实施：** **绝对不在 8GB 虚拟机上部署本地 LLM。** 核心的大语言模型推理（如 Qwen、DeepSeek）以及高消耗的重排序（Reranker）模型，全部通过调用云端 API 实现。将本地宝贵的内存资源全部分配给数据库、后端逻辑和前端可视化界面。

### 3.2 组件降维：使用轻量级平替技术栈

- **策略：** 放弃企业级重型数据库，选用低开销方案。
- **实施：**
	- **向量库：** 弃用 Milvus，改用无服务依赖的 `ChromaDB` 或基于 SQLite 的 `SQLite-VSS`，直接以文件形式运行，内存占用极低。
	- **集成框架：** 可采用轻量级开源框架（如 LlamaIndex 或定制化的 LangChain 脚本）串联流程，避免引入过于庞大的系统中间件。

### 3.3 时空转换：离线构建，在线查询

- **策略：** 将高耗时的知识处理工作前置。
- **实施：** 知识切分、向量化 embedding 生成、以及**最耗时的知识图谱实体抽取**，全部在开发团队的高性能物理机上“离线”完成。比赛部署时，虚拟机内只存放清洗干净、已经建好的轻量级数据库文件。系统在演示期间**只读不写（或仅做轻量级追加）**，彻底消除构建图谱时的计算瓶颈。

### 3.4 降级备用预案：保底的用户体验

- **策略：** 防范演示翻车。
- **实施：** 在混合检索模块加入超时控制。如果图谱查询或向量查询在 5 秒内未响应，系统自动降级为纯文本关键词匹配，先给出基础的安全操作手册，确保“任何情况下系统都不崩溃、不卡死”，保住“用户体验”和“稳定性”评分项。
