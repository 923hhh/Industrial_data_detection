# TodoSoftBei：软件杯赛题适配整改清单

## 目标

围绕第十五届中国软件杯赛题《基于多模态大模型技术的设备检修知识检索与作业系统》，将当前“工业故障诊断原型”重构为“设备检修知识与作业助手”，并按比赛评分维度推进功能、展示、证据和材料补齐。

## 赛题依据

- 赛题名称：`A基于多模态大模型技术的设备检修知识检索与作业系统`
- 组别：`A组`
- 出题企业：`龙芯中科技术股份有限公司`
- 赛题页：`https://www.cnsoftbei.com/content-15-1246-1.html`
- 本轮排除项：`LoongArch + 银河麒麟高级服务器版` 适配暂不纳入当前整改主线，仅在风险中保留说明。

## 当前结论

- 当前仓库已经具备 FastAPI、PostgreSQL、SSE、多智能体诊断和真实部署能力，但主体仍是“工业故障诊断原型”。
- 当前项目与赛题仍存在明显定义偏差：赛题主体是“检修知识检索 + 标准化作业 + 知识沉淀更新”，现有系统只覆盖了“智能分析子模块”的一部分。
- 当前优先级不是继续打磨旧诊断页，而是先冻结作品定义、知识库边界和演示主线，再开展赛题功能建设。

## 三个月大改增补项（2026-03-30）

### 第二阶段增强项：把知识中心做成更完整的业务入口

- OCR：
  - 支持扫描件或图片型 PDF 先做 OCR，再进入知识导入与检索链路。
  - 优先目标：至少支持“扫描件 PDF -> 预览 -> 导入”最小闭环。
- 图片上传：
  - 正式前端知识中心增加稳定的图片上传入口，而不是只保留联调级单图输入。
  - 与知识导入、知识检索两条路径区分：一条用于文档导入，一条用于故障图检索。
- 多模态问答：
  - 在知识检索中心补“文本 + 型号 + 图片”的联合问答卡片，输出引用支撑下的回答，而不是只返回命中列表。
  - 问答仍需以知识引用为准，不走纯大模型自由回答。
- 工单/设备信息：
  - 在正式前端和后端任务入口中补充设备编号、设备型号、工单编号、报修来源、优先级等业务字段。
  - 目标是让“检修任务”更像真实工单，而不是单次演示任务。
- 检修步骤输出：
  - 在知识问答和任务生成之间增加明确的“检修步骤输出”层。
  - 输出应结构化包含步骤编号、工具/材料、注意事项、风险提醒和确认动作。

> 当前进展（2026-03-30）：
> - 已落地 OCR 最小闭环：知识中心支持图片型知识导入预览、正式图片导入和处理提示展示。
> - 已落地正式图片上传入口：知识检索中心支持图片预览、图片分析结果、有效检索词和 OCR/视觉提示展示。
> - 当前仍待补的是“扫描件 PDF 专项处理”“导入前人工校对”“工单/设备信息扩展”和“结构化检修步骤输出”。

### 第三阶段亮点项：把 Agent 和知识能力升级成答辩亮点

- 混合检索：
  - 在当前全文检索基础上，补 BM25/全文检索 + 规则召回 + 案例召回的混合策略。
  - 目标是同时兼顾通用手册、标准步骤和历史案例。
- rerank：
  - 在候选结果召回后增加重排层，优先返回与当前设备型号、故障现象和工单上下文更相关的知识条目。
- 相似案例推荐：
  - 除知识条目外，增加“类似案例”推荐卡片，展示相似工况、处理步骤和最终结果。
- 安全提醒：
  - 将安全提醒从静态文案升级为独立能力，按设备类型、故障类型和作业步骤动态生成。
- 检修流程推荐：
  - 在 `WorkOrderPlannerAgent` 基础上进一步补“流程推荐”能力，让系统不仅输出步骤，还能推荐适合当前情境的标准检修路径。
